\section*{Company Overview}
AlturaSoft builds and operates the Altura Cloud Platform (ACP) and Altura InsightHub, serving enterprise customers globally. In addition to our SaaS products, we deliver Professional Services for implementations, integrations, and managed SRE operations.

\begin{itemize}[leftmargin=*, label=\textbf{•}]
    \item \textbf{Products:} Altura Cloud Platform (multi-tenant SaaS), Altura InsightHub (analytics \& dashboards).
    \item \textbf{Service Lines:} Solutions Engineering (pre-sales), Customer Implementations, Managed Services (SRE-as-a-Service).
    \item \textbf{Primary Tech Stack:} AWS, Kubernetes, Terraform, Go, Java, TypeScript, Python, PostgreSQL, Kafka, Snowflake, GitHub Actions, Datadog, Vault.
    \item \textbf{Compliance \& Security:} SOC 2 Type II and ISO 27001 programs; SSO via Okta; MFA enforced; least-privilege access; secrets in Vault.
\end{itemize}

\section*{Engineering Practices \& Standards}
We practice agile delivery with two-week sprints, trunk-based development, and mandatory code review (2 approvals). DORA metrics are tracked across teams; services must pass production-readiness reviews before launch.

\paragraph*{Definition of Done:} Code + tests + docs; SLOs defined; dashboards \& alerts; runbooks; rollback plan; security review.
\paragraph*{Branching \& CI/CD:} Trunk-based with short-lived feature branches; CI gates for tests/security/coverage; canary and progressive delivery.
\paragraph*{Observability:} Standardized RED/USE dashboards; SLO/error budgets enforced via SRE review.
\paragraph*{Data Governance:} Data classification, lineage, and PII handling policies; PR checks for schema changes.

\section*{Career Framework (Engineering)}
\paragraph*{Individual Contributor (IC) levels and typical scope:}
\begin{itemize}
    \item Associate Engineer (IC1): Learns codebase; delivers small, well-defined tasks with mentorship.
    \item Software Engineer (IC2): Owns small features; contributes to reviews and test quality.
    \item Senior Engineer (IC3): Leads features; mentors; designs within a service boundary.
    \item Staff/Principal Engineer (IC4-IC5): Cross-team design; sets standards; deep domain leadership.
    \item Distinguished/Architect (IC6): Broad, company-wide technical strategy.
\end{itemize}
